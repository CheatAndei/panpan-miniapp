# 番番记录熟人老师共用版改造方案

更新日期：2026-07-08

## 1. 定位

`番番记录` 名字保持不变。

当前目标不是做公开平台，也不是做商业化系统，而是把现有私人教培小程序改造成：

> 给自己和关系好的几个老师一起使用的课堂记录、家校反馈、课表签到小工具。

核心边界：

- 不碰收费。
- 不接支付。
- 不做订单。
- 不做平台抽佣。
- 不做复杂机构后台。
- 不做大型 SaaS。
- 优先服务熟人老师小范围使用。

## 2. 改造目标

### 2.1 对老师

让每个老师都能独立使用：

- 管自己的班级。
- 管自己的学生。
- 排自己的课表。
- 发自己的课后反馈。
- 处理自己学生的请假。
- 家长看到的是对应老师的信息。

### 2.2 对家长

让家长端保持简单：

- 微信登录。
- 绑定孩子。
- 看课表。
- 看签到状态。
- 看课后反馈。
- 看老师印象/学生画像。
- 提交请假。
- 接收通知。

### 2.3 对产品

保留 `番番记录` 作为产品名，但弱化“潘潘老师专属”的感觉。

建议关系改为：

- `番番记录` = 小程序名字。
- `潘潘老师` = 其中一个老师。
- 其他老师也可以配置自己的老师名称。

## 3. P0：必须先做

### 3.1 文案去个人专属化

把固定的个人化表达改成通用表达。

示例：

```text
原：潘潘老师课堂管理助手
改：番番记录｜课堂记录与家校反馈助手
```

```text
原：欢迎使用潘潘老师课堂管理助手
改：欢迎使用番番记录
```

```text
原：潘潘老师，今天有 3 节课
改：{老师称呼}，今天有 3 节课
```

注意：

- 产品名不改，仍是 `番番记录`。
- 老师称呼根据当前登录老师资料显示。
- 家长端根据孩子所属老师显示对应老师称呼。

### 3.2 老师资料配置

每个老师需要有自己的基础资料。

建议字段：

- 老师显示名，例如 `潘潘老师`、`李老师`。
- 头像，可先用默认头像或首字头像。
- 科目，例如数学、英语、物理。
- 简短介绍。
- 联系方式，可选。
- 是否启用。

老师端需要一个简单设置入口：

```text
我的 -> 老师资料
```

### 3.3 老师数据隔离

几个老师一起使用时，数据不能串。

最低要求：

- 老师只能看到自己的班级。
- 老师只能看到自己的学生。
- 老师只能看到自己的课表。
- 老师只能发布自己的反馈。
- 老师只能审批自己学生的请假。
- 家长只能看到已绑定孩子的信息。

现阶段不需要复杂权限系统，先保持“一位老师一个独立空间”。

### 3.4 家长端显示对应老师

家长绑定孩子后，家长端应该展示孩子对应老师的信息。

例如：

- 课表页显示对应老师。
- 反馈页显示对应老师。
- 老师印象页显示对应老师。
- 我的页面显示已绑定孩子和老师。

避免所有家长都看到固定的“潘潘老师”。

### 3.5 老师开通方式保持简单

不做公开注册审核。

建议保持熟人模式：

- 普通微信登录默认是家长。
- 老师需要输入教师邀请码。
- 你手动把邀请码发给关系好的老师。
- 输入正确后升级为老师角色。

这样安全、简单、可控。

## 4. P1：让熟人老师更容易用

### 4.1 一键分享绑定孩子

当前邀请码可以保留，但要减少老师操作成本。

建议：

- 学生列表中增加 `分享给家长`。
- 分享卡片自动带学生邀请码。
- 家长打开后进入绑定确认页。
- 邀请码自动填入。
- 家长确认后绑定孩子。

优先做微信分享卡片，二维码可以后做。

### 4.2 新老师上手引导

老师第一次进入时展示 5 步引导：

1. 设置老师资料。
2. 创建第一个班级。
3. 添加学生。
4. 分享绑定邀请给家长。
5. 创建课表并开始使用。

每一步都给一个按钮，直接跳转对应页面。

### 4.3 老师首页今日待办

老师首页建议展示：

- 今日课程。
- 待签到学生。
- 待签退学生。
- 待发布课后反馈。
- 待处理请假。

目标是让老师打开就知道今天要做什么。

### 4.4 家长首页更清晰

家长首页优先展示：

- 今日是否有课。
- 最新签到/签退状态。
- 最新课后反馈。
- 请假入口。
- 孩子当前老师。

家长端不要复杂，信息越直接越好。

## 5. P2：提升专业感

### 5.1 学生成长时间线

以孩子为中心展示长期记录：

- 上课记录。
- 签到记录。
- 请假记录。
- 课后反馈。
- PDF 学习笔记。
- 老师印象/学生画像。

这会让家长觉得记录是连续的，不只是一次性通知。

### 5.2 阶段性学习总结

基于最近几次反馈，生成阶段总结：

- 最近学习状态。
- 明显进步。
- 需要加强的点。
- 老师建议。

先给老师生成草稿，老师确认后再给家长看。

### 5.3 AI 反馈模板

AI 功能可以继续保留，但要更适合不同老师。

建议模板：

- 简短反馈。
- 详细反馈。
- 鼓励型反馈。
- 严谨型反馈。
- 低年级反馈。
- 初高中反馈。

重要原则：

- AI 只生成草稿。
- 老师确认后才发布。
- 避免负面标签。
- 避免过度判断孩子。

### 5.4 家长已读状态

老师发反馈后，可以看到家长是否已读。

不用做复杂互动，先只做：

- 未读。
- 已读。
- 已读时间。

## 6. 暂时不做

以下内容当前阶段不做：

- 微信支付。
- 订单系统。
- 课包购买。
- 收费套餐。
- 平台抽佣。
- 多机构后台。
- 多校区管理。
- 财务统计。
- 公开招生报名。
- 平台级营销系统。
- 复杂 RBAC 权限系统。

## 7. 推荐开发顺序

### 第一步：梳理写死文案

找出项目中所有固定写死的：

- 潘潘老师。
- 课堂管理助手。
- 固定老师头像。
- 固定老师介绍。

决定哪些保留为默认值，哪些改成当前老师资料。

### 第二步：补老师资料字段

在后端增加老师资料能力，前端增加设置页或设置入口。

最小版本只需要：

- 显示名。
- 科目。
- 简介。
- 头像或默认头像。

### 第三步：改家长端老师展示

家长绑定孩子后，家长端所有老师相关内容都从孩子所属老师资料读取。

### 第四步：确认数据隔离

逐个检查老师端接口：

- 班级。
- 学生。
- 课表。
- 签到。
- 反馈。
- 请假。
- 学生画像。

确保都按当前老师过滤。

### 第五步：做分享绑定

把手动复制邀请码升级成一键分享。

### 第六步：做新老师引导

降低关系好的老师第一次使用成本。

## 8. 成功标准

这个阶段做到下面几点就算成功：

- 关系好的老师可以自己登录并成为老师。
- 老师能配置自己的老师名称。
- 老师只能看到自己的数据。
- 家长绑定孩子后看到的是对应老师。
- 老师可以顺利完成：建班 -> 加学生 -> 邀请家长 -> 排课 -> 签到 -> 发反馈。
- 家长可以顺利完成：登录 -> 绑定孩子 -> 看课表 -> 看签到 -> 看反馈 -> 请假。
- 整个产品没有收费、支付、订单相关设计。

## 9. 产品口径

对外可以这样介绍：

```text
番番记录是一个课堂记录和家校反馈小工具，适合老师记录上课、签到、请假和课后反馈。现在先给熟悉的老师小范围使用，每个老师的数据相互独立。
```

避免使用这些说法：

- 平台。
- SaaS。
- 商业化。
- 收费系统。
- 机构管理系统。
- 支付闭环。

当前阶段更适合说：

- 小工具。
- 熟人内测。
- 课堂记录。
- 家校反馈。
- 老师独立空间。
