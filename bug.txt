Bug3.0 修复任务书
更新日期：2026-07-05

目标：
只处理 bug3.0。先修可复现 bug，再做体验优化。暂不处理 bug1.0 / bug2.0 历史项。

执行原则：
1. 先修签到、请假、课程发布、通知、反馈、PDF 主链路。
2. 每个点修完要能验证。
3. 不部署，部署前单独确认。
4. 不重构账号体系，不做数据库大迁移。
5. UI 美化放在稳定性 bug 后面。

一、P0 先修

P0-1 签到 / 签退 / 请假状态闭环
状态：已修，已通过构建
问题：
- 家长端需要看到签到、签退提醒时间。
- 签到页需要请假按钮。
- 老师在签到页标记请假后，家长端对应显示已请假。
- 课后反馈中不出现已请假学生。
验收：
- 老师签到后，家长端显示“今日已签到 HH:mm”。
- 老师签退后，家长端显示“今日已签退 HH:mm”。
- 老师标记请假后，家长端显示“今日已请假”。
- 已请假学生不能再签到。
- 反馈页不显示已请假学生。

P0-2 课程发布逻辑支持多次上课
状态：已修核心逻辑，假期连续批量发布另拆功能
问题：
- 假期连续上课时，同一学习小组可能多天、多次上课。
- 当前按周几发布，容易错发、漏发、重复。
验收：
- 同一学习小组可在不同日期发布多次课程。
- 重复发布同一 schedule_id + class_date 时跳过。
- 特殊发布同班同日同时间时不重复插入。
- 签到页只显示真实已发布课程。

P0-3 课程发布无通知
状态：已修，已通过构建
问题：
- 发布课程后家长端没有通知。
验收：
- 发布课程后调用上课提醒。
- 模板未配置、openid 缺失、微信接口失败时有明确提示。
- 前端能显示“已发布并提醒”或“已发布但提醒未送达”。

P0-4 课后反馈 PDF 发送失败
状态：已修上传兼容，已通过构建
问题：
- 老师上传 / 发送 PDF 后，家长端可能打不开。
验收：
- 上传 PDF 成功。
- 发布反馈和单独发送学习笔记都能带 PDF。
- 家长端首页、反馈页都能打开 PDF。
- 失败时有明确提示。

二、P1 功能修补

P1-1 特殊老师签退
状态：待处理
需求：
- 签到页除普通签退外，增加特殊老师签退。
- 文案：xx老师已离开，请家长主动联系小朋友沟通安排后续。
- xx 可在“我的”页面编辑。
验收：
- 老师可设置显示称呼。
- 特殊签退不影响普通签退。
- 家长端能看到特殊签退文案。

P1-2 请假审批接收家长意见反馈
状态：待处理
需求：
- 请假审批页展示家长提交的意见 / 原因。
- 拒绝请假时，拒绝理由不能默认成“好的”。
验收：
- 家长意见完整展示。
- 老师回复为空时不写默认无意义内容。

P1-3 课后反馈可手动输入
状态：待处理
需求：
- AI 生成失败时，老师可以手动输入。
验收：
- 学习小组总反馈可手动输入并发布。
- 学生个人反馈可手动输入并发布。
- 不强制必须先 AI 生成。

P1-4 学生反馈格式约束
状态：待处理
格式：
- 第一行：学生名字 + emoji。
- 第二行开始写中文反馈。
- 中文段落缩进 2 格。
- 可多分段，每段都缩进 2 格。
验收：
- 老师端编辑时格式可控。
- 家长端展示不乱。

P1-5 写反馈展开 / 收回
状态：待处理
需求：
- 点击“写反馈”展开。
- 再次点击上方空白处可收回。
验收：
- 收回时不丢已输入内容。
