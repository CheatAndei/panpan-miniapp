## 1. 目标

把当前以七年级数学为主的学习中心，升级为：

- 七年级、八年级、冲刺中考三个自由切换入口。
- 题图清晰、连续闯关、知识点刷题、多学科试卷库。
- 计算题符号书写规范，学生能一键报错，老师能复核、停题和追踪处理。
- 课后反馈可生成清晰的个人反馈卡，支持单人和全班保存，同时消除无出门测提示与展开态误删。
- 游客体验 -> 联系老师 -> 绑定学生 -> 完成练习 -> 生成成就海报 -> 带来新游客的增长闭环。

不改变现有 Vue 3 + uni-app、Express、Docker 主技术栈；不一次性重写。

## 2. 已核实现状

### 2.1 代码能力

- `classes.grade/subject`、`students.grade` 已有字段，但只是自由文本。
- 每日练习硬编码为初中数学；口算王只有小学/初中两档。
- 试卷表有 `grade`，没有 `subject`；考试类型只允许期中、期末、月考。
- 试卷、选择刷题王、压轴挑战资源和导入脚本均含 `GZ7`/七上硬编码。
- 选择刷题王题干与真实选项主要依赖图片；图片小会直接导致无法答题。
- 压轴挑战按 `student_id + week_start` 唯一，只能每周一题。
- 当前无成就海报、动态小程序码、游客练习隔离能力。
- 所有学习功能要求已登录家长绑定学生；试卷文件也受绑定关系保护。
- 已核查口算王与每日练习四套生成资源，共 2720 道题。确认 108 道题存在不规范正负号连写。
- 口算王初中题库有 84 道 `1 + -2` 一类题干：分数口算 41 道、小数口算 43 道。
- 每日练习当前启用的 `junior-calculation-v3` 有 24 道 `0.2x+-2.2` 一类一元一次方程。
- 根因是不同生成器没有统一经过带括号的有符号数格式化。选择刷题王已有报错闭环，口算王和学习中心计算题尚无报错入口、API 和教师处理队列。

## 3. 产品信息架构

### 3.1 统一学习入口

1. 七年级
2. 八年级
3. 冲刺中考

规则：

- 小学生、其他年级、游客都能切换。
- 路由统一携带 `grade=g7|g8|g9`。
- 每个学生保存最近选择；首次进入时优先识别学生年级，无法识别则默认七年级。（已确认）
- 分享链接指定年级时，以链接年级优先。

| 入口 | 首期内容 | 后续内容 |

## 4. 需求方案

### 4.1 选择刷题王题图

新增公共题图阅读组件 `pp-question-reader`：

- 题图脱离普通白卡，接近全屏宽。
- 提供“适应屏幕 / 放大阅读”切换。
- 放大阅读约 1.5-1.7 倍，可横向滑动。
- 点击进入微信原生大图预览，提示“双指缩放”。
- 题号、来源、答案按钮不跟随图片横向移动。
- 记住用户阅读模式。
- 审计异常裁图；包含相邻题目的图片重新裁切。

验收：320、375、430px 等效宽度与微信真机均能读题；横滑不误触答案。

### 4.2 压轴挑战题图

- 复用同一题图阅读组件。
- 私有图片继续走鉴权下载。
- 保留加载、失败、重试、预览状态。
- 长题默认放大阅读，避免整题压成小字。

验收：复杂几何图、长题干、横向公式清晰；未领取题目者无法直接访问图片。

### 4.3 七、八、九年级入口

建立统一内容维度：

- `grade_code`: `g7/g8/g9`
- `subject_code`: `math/chinese/english/physics/chemistry/history/morality`
- `semester_code`
- `exam_type`
- `topic_key`
- `difficulty`

改造范围：学习中心、题库 API、试卷库、选择题、压轴挑战、统计与分享。

原则：年级只筛内容，不校验用户是否“有资格进入”。

### 4.4 冲刺中考全科一模卷库

首期只做整卷资料库，不自动拆题。（已确认）

页面筛选：

- 年份：2023/2024/2025/2026
- 学科：七科
- 地区：广州各区/市级
- 文件：题目版/解析版
- 状态：配对完整/缺解析

导入流程：

1. 规范政治/道德与法治等别名。
2. 题目版与解析版配对。
3. 哈希去重。
4. 校验文件可打开、页数、大小。
5. DOCX 审核后转 PDF；原 DOCX 可保留教师下载。
6. 先导入一个年份、一个学科作为试点。
7. 人工验收后批量发布。

代码基础改造：

- `exam_papers` 增加学科、地区标准码、资源类型。
- 扩展考试类型为一模。
- 稳定编号从 `GZ7-*` 改为通用编号。
- 重写文件配对逻辑；当前脚本按目录只选一对文件，不适合新资料目录。
- API 支持年级、学科、地区、年份组合筛选。

存储：539 MiB 不进入小程序包。长期建议对象存储；若首期继续 Docker volume，发布前要求服务器至少预留数倍空间、备份和下载监控。

### 4.5 八上知识点板块

建议产品名：`八上知识点闯关`。首期只做知识点闯关，暂不上八年级试卷与压轴挑战。（已确认）

不展示原 PDF；把资料转为知识结构和原创选择题：

1. 章节 -> 知识点 -> 易错点 -> 难度。
2. 每个知识点包含简短知识卡、8 题闯关、解析、错题复练。
3. 单局建议：3 基础 + 3 核心 + 2 易错。
4. 正确率达到门槛标记掌握；错题按间隔复现。
5. 几何图单独出图，正文使用可重排文字。

首期：60-100 道人工审核题，覆盖 6 章、3 专题。流程：

`提取知识点 -> 原创出题 -> 干扰项审查 -> 公式/图形校验 -> 教师审核 -> 小范围试用 -> 发布`

不建议 AI 一次生成大量题后直接上线。

### 4.6 压轴挑战改为连续闯关

取消所有“一周一次、本周、周起始日”产品文案与业务限制。

状态机：

`领取题目 -> 作答上传 -> 等待批改 -> 判错后重交同题 / 判对后解锁下一题`

规则建议：

- 每名学生每个年级、题型只有一道进行中题目。
- 判错：保留批改历史，允许不限次数重新上传。（已确认）
- 判对：记录通关，由用户手动点击“下一题”。（已确认）
- 新题从对应年级、题型的未通关题库全随机抽取，不设难度顺序。（已确认）
- 未提交前每天允许更换 1 次题目；提交后、等待批改时、判错重交时不能换题。
- 被更换的题目回到未通关题池，之后可能再次抽到，但不得立即重复。
- 不重复发已通关题，题库耗尽后明确提示。
- 老师可“判对、判错、跳过异常题”。

数据库建议新建 V2 连续挑战表，迁移旧历史；不要强改现有 weekly 唯一约束。提交记录允许多次尝试，并保留每次附件与批语。

### 4.7 三类成就海报

共用海报引擎，数据必须来自服务端真实成绩。海报包含品牌、小程序码、入口参数、隐私安全文案。

压轴挑战：

- 触发：老师判对。
- 文案示例：`拿下海珠区 2025 一模数学压轴题`、`谁来挑战这道题？`
- 展示原题局部，不显示答案；学生名统一显示为“姓 + 同学”，例如“潘同学”。（已确认）

口算王：

- 触发：完成有效挑战。
- 展示正确率、用时、得分、真实排名/超过比例。
- 样本过小时不展示百分位。

选择刷题王：

- 正确里程碑初始配置：30/100/300/500/800/1000/1200/1500/2000/3000/5000。
- 完成里程碑初始配置：50/100/200/300/500/800/1000/1200/1500/2000/3000/5000。
- 里程碑必须使用可配置表，不在前端硬编码；题库扩充后可继续增加。
- 同次作答跨过多个里程碑时只弹一张合并成就卡，其余成就进入成长记录，避免连续弹窗。
- 展示覆盖试卷数量和部分真实来源。
- 每个里程碑只弹一次，可从成长记录重新生成。

技术路径：

- 后端生成动态小程序码，`scene` 只放不可枚举活动码。
- 海报可由后端 `sharp` 合成，返回受控图片；客户端预览、保存相册、分享好友。
- 微信不能替用户自动发朋友圈；产品文案应写“保存海报分享到朋友圈”。
- 分享落地页直接进入对应游客体验。

### 4.8 游客体验与转化

不新增 `guest` 业务角色。把“已微信登录但未绑定学生”的用户定义为体验用户。

游客可用：

- 选择题不限体验次数。
- 口算不限体验局数。
- 查看一张压轴题示例与交互说明；不进入教师批改队列。
- 游客成绩不进真实排行榜、学生成长记录和错题复习；服务端仍设置基础频率限制，防止恶意请求。

游客禁止：

- 打开或下载试卷。
- 查看解析卷。
- 进入真实排行榜。
- 写入真实学生成长、错题、班级数据。

转化提醒只放三个节点：

1. 首页游客状态条。
2. 体验完成结果页。
3. 点击试卷或超出体验额度时。

统一 CTA：`联系潘潘老师加入`。

已确认采用复制微信号方案：

- 页面显示：`联系潘潘老师，微信号 sysu0011203`。
- 用户主动点击“复制微信号”，调用系统剪贴板能力。
- 不强制添加，不把“添加老师”作为获得奖励或继续使用免费功能的条件。
- 小程序不提供姓名、手机号、用户微信号收集表单。
- 复制老师公开联系方式不等于收集用户个人信息。
- 直接展示个人微信号仍可能存在审核风险；微信审核结果不可保证。联系方式必须做成配置项，若审核不通过，可快速切换为微信官方客服按钮。

### 4.9 口算/学习计算题符号修复与题目报错

先修根因，再增加学生报错入口；不能用报错按钮代替题库质量检查。

符号修复：

- 修复已确认的 108 道题：口算王 84 道改为 `1 + (−2)` 一类写法；每日练习 24 道改为 `0.2x − 2.2` 一类规范方程。
- 新增统一表达式格式化函数；显示层使用清晰的数学负号 `−`，答案与计算数据仍使用可解析数值。
- 对已核查的全部 2720 道生成题做自动扫描，禁止裸露的 `+ -`、`- -`、`+-`、`--`。
- 题库生成或启动校验不通过时禁止发布，测试输出题目 ID、题型和题干。
- 口算题使用旧题干作为 `idStem`，可修改显示题干而保持稳定 ID；每日练习使用显式题干迁移并严格限定允许变化，保留原题 ID、签名与历史关联。
- 现有 `mental_challenges`、`learning_attempts` 都保存了题目快照。接口对旧快照生成规范化 `display_stem`，原始题干继续保留用于审计和历史复核。

学生端：

- 口算王、学习中心计算题的作答页，每题下方增加低干扰按钮 `题目有问题`；两个结果页每道回顾题也能补报。
- 原因选项：`正负号/括号错误`、`题目条件不清`、`答案有误`、`重复题`、`其他`；可补充不超过 200 字说明。
- 自动携带来源类型、挑战/练习 ID、题目 ID、题型、题干快照、当前题号、学生、家长和提交时间，不要求学生手抄题目。
- 报错不直接修改答案、分数或排行榜；同一学生同一次练习同一题只保留一条待处理记录，每个家长每小时最多提交 5 条。
- 报错面板不暂停计时，防止利用报错暂停刷成绩；结果页保留补报入口，避免学生赶时间时无法反馈。（已确认）

教师端：

- 把现有“选择题报错处理”升级为统一“题目报错处理”，按 `选择题 / 口算题 / 学习计算题` 和 `待处理 / 已修复 / 不成立` 筛选。
- 教师只能看到自己学生提交的记录，可填写处理说明并标记处理状态。
- 确认题目有误时，可停用题目：口算题写入停题控制表，练习题使用现有 `practice_questions.is_active`；后续抽题立即排除，历史快照不删除。
- 同题出现 3 名不同学生报错时生成高优先级提醒，但仍由老师确认，不自动停题。
- 若老师确认题干或答案影响作答，生成受影响挑战清单和成绩修正记录；不得因一次未核实反馈直接改分。

数据与接口：

- 新建 `calculation_question_reports`，同时支持 `mental_challenge/learning_attempt`，保存题目快照、原因、处理状态和处理人；不能只外键依赖当前题库。
- 新建 `mental_question_controls`，保存停用题目 ID、原因、操作人和时间，抽题时统一过滤。

- 前端只提交来源 ID、题目 ID、原因和说明；题干、标准答案等快照必须由服务端读取，提交成功响应不能向未交卷学生泄露标准答案。
- 复用选择题报错的去重、限频、教师归属和审计原则，不把用户填写内容写入公开日志。

验收：2720 道生成题无不规范连写；旧挑战、旧学习任务显示正常；越权、重复、限频、停题、教师迁班归属和历史快照测试全部通过。

### 4.10 个人反馈静态图片

建议产品名：`个人反馈卡`。与公开传播的成就海报分开：反馈卡含学生个人内容，仅教师生成、保存手机后私发家长；家长端不增加静态卡，不放小程序码，不进入公开分享链路。（已确认）

卡片内容：

- 固定 1080×1440、3:4 竖图；暖白底、深绿文字、少量金色点缀，与现有小程序一致。
- 顶部：`潘潘课堂反馈`、课程日期、课程主题、学生姓名。
- 中部：学生反馈正文；按实际字数换行，字号设最低值，超过安全长度时阻止导出并提示老师精简，不能自动缩成小字。
- 图片：0 张时使用留白；1 张大图；2 张并排；3 张一大两小。统一 `aspectFill`，导出前可预览裁切，不能拉伸人脸或作业照片。
- 右下角放 02 线条风潘潘老师形象，先裁成头肩半身、压缩为小体积本地资源；不能直接把 2.21 MiB 原图放进小程序包。
- 底部：`潘潘老师 · 课堂观察` 与 `仅供学生家长查看`；默认不显示微信号、二维码、学校和班级。

操作设计：

- 每名有反馈、未请假的学生卡内增加 `预览反馈卡`、`保存此同学`。
- 学生列表顶部增加 `保存全部反馈卡`：依次保存多张独立个人卡，不生成包含全班姓名的一张长图。（已确认）
- 保存全部前检查：所有未请假学生均已有反馈；缺失时列出姓名，不静默跳过。
- 批量过程显示 `正在生成 3/12`；逐张串行生成、串行保存，结束显示成功/失败数量，并支持只重试失败项。
- 首次保存时由用户主动触发相册授权；拒绝后给出重新授权指引。生成文件必须先成为本地临时图片，网络图片不能直接保存到相册。
- 保存操作不自动发布反馈；发布与保存保持两个明确动作，避免老师误以为保存等于已通知家长。

技术建议：

- 首期使用小程序端隐藏 Canvas 生成本地临时 PNG：现有学生图片仍在教师设备，减少新的服务端个人图片副本与数据库字段。
- 新增公共 `utils/feedback-card.js`，负责文字折行、图片布局、IP 形象、高清导出、预览和保存；页面只负责编排操作。
- 远程历史图片先下载成临时文件再绘制；单张图片失败时允许无图重试或跳过，不能整班卡死。
- 中文使用系统可用字体；反馈正文中的 emoji 转为固定小图标或移除，避免不同手机渲染为空框。
- 若以后需要家长端长期查看/下载反馈卡，再升级为后端 `sharp` 私有生成与受控文件；首期不做。

隐私：反馈卡使用学生全名。（已确认）因此只适合私发；若可能发朋友圈，必须另设“匿名版”，显示“姓 + 同学”并移除学生照片，不能复用个人反馈卡直接公开。

### 4.11 无出门测时保持沉默

- 关闭 `本次有出门测` 后，只依据课堂表现、老师备注和课程内容生成反馈。
- 页面提示改为：`本次不使用出门测信息，反馈只写课堂表现。`
- 批量与单人提示词均禁止出现“出门测、测试、未安排、未进行”等说明；不得猜测测试表现。
- 服务端增加兜底清理与输出校验。模型仍返回相关句子时删除该句；删除后内容为空则返回生成失败，要求重试。
- 有出门测时维持现有逻辑：可参考大致水平，但最终反馈不出现具体分数。

验收：关闭出门测后，批量、单人、简洁、温馨四种组合的最终反馈均不出现是否测试的说明。

### 4.12 展开反馈后禁用右滑删除

- 仅折叠状态允许右滑露出删除按钮。
- 点击 `写反馈` 时立即关闭该卡已有滑动状态；展开态不绑定删除滑动手势。
- 展开态的滑块、输入框、图片预览、移除图片和纵向滚动不触发卡片位移。
- 收起后恢复右滑；其他仍折叠的反馈卡不受影响。
- 即使露出删除按钮，也继续保留二次确认；发布中的卡片禁止删除。

验收：展开态连续操作课堂表现滑块、出门测滑块、输入框和图片 50 次，不露出删除按钮；折叠态滑删与确认框正常。

## 7. 必测项

- 七年级原功能无回退。
- 年级自由切换，不产生越权或混合统计。
- 游客无法通过直接 URL/API 下载试卷。
- 答错可重交；答对只解锁一次；并发点击不重复发题。
- 题目、答案、解析不串题。
- 口算与学习题库不存在不规范连续正负号；新旧挑战、学习任务题干均能正确显示。
- 学生只能报自己挑战/学习任务中的题；重复报错与恶意高频提交被拦截。
- 教师只能处理自己学生的报错；停题后新挑战不再抽到该题，历史记录仍可复核。
- 关闭出门测后，所有个人反馈均不提是否测试。
- 展开反馈卡后无法滑出删除按钮；折叠后可正常滑删并二次确认。
- 单人反馈卡在主流 iOS/Android 真机文字不截断、图片不变形、IP 不遮挡正文。
- 全班保存不会漏人、重复保存或因单张失败中断；拒绝相册授权时有可恢复提示。
- 海报指标真实；同一里程碑不重复弹。
- 小程序码落地到正确年级和功能。
- 海报不泄露真实姓名、学校、班级、手机号。
- PDF/DOCX 可打开，题解配对正确，缺失状态明确。
- 数据迁移前后表记录数、附件引用、历史成绩一致。

## 8. 主要风险

| 风险 | 控制 |
|---|---|
| 第三方资料授权 | 未确认前仅内部使用；八上只参考并原创出题 |
| 数学公式/OCR 错误 | 公式人工录入，题目教师审核后发布 |
| 自动生成题符号或答案错误 | 统一格式化器、全题库静态审计、学生报错与教师停题闭环 |
| 本地文件持续增长 | 对象存储优先；磁盘、备份、带宽设阈值 |
| `sql.js` 复杂迁移 | 备份、dry-run、数量校验、回滚脚本 |
| 游客污染成绩 | demo 独立会话，不进学生表与排行榜 |
| 海报夸大或隐私泄露 | 只用真实数据；默认匿名/昵称；服务端签名 |
| 个人反馈卡泄露学生信息 | 默认仅私发；全班按独立图片保存；公开版必须匿名且移除学生照片 |

| Canvas 字体/图片跨机型异常 | 固定尺寸、最低字号、图片预下载、emoji 归一、iOS/Android 真机验收 |
| 分享弹窗骚扰 | 里程碑一次性触发，可关闭、可稍后生成 |
